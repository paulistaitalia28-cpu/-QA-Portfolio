# QA Portfolio — Fernanda Gomes  **Analista de QA formada pela TripleTen (2026)** 
 Este repositório reúne os principais projetos que desenvolvi durante o **Bootcamp Analista de QA da TripleTen** (330h, janeiro a junho de 2026). Cada pasta representa uma sprint do curso e contém casos de teste, evidências de execução, bug reports e relatórios técnicos. 
  **O que você vai encontrar aqui:** - Testes manuais estruturados (BVA, classes de equivalência, testes exploratórios) - Testes cross-browser (Chrome, Firefox, Opera) - Testes de API com Postman - Automação Web com Selenium + Python + Pytest - Bug reporting profissional no Jira e GitHub Issues 
  ## Cases documentados  ### 📁 [Sprint 1 — Regressão Urban Routes](./sprint1-regressao-urban-routes/) Execução de suite de regressão em plataforma web. Casos de teste + relatórios de bug documentados. 
   ### 📁 [Sprint 3 — Testes Web Cross-Browser](./sprint3-testes-web-cross-browser/) Aplicação de técnicas de particionamento por classes de equivalência. Bugs cross-browser documentados. 
   ### 📁 [Sprint 4 — Testes de API (Order and Go)](./sprint4-teste-api-order-and-go/) Coleção Postman com testes POST/DELETE e validação de contratos. 
   ### 📁 [Sprint 8 — Automação Selenium + POM](https://github.com/paulistaitalia28-cpu/QA-Brazil_Python_Automation) Automação Web em Python + Selenium + Pytest, seguindo Page Object Model. **Código completo no repositório linkado.** 
   ## 📁 [Sprint 9 — Projeto Final](./sprint9-projeto-final/) Testes Web + Mobile + API. Casos de teste projetados com BVA e classes de equivalência, bugs descobertos e documentados no Jira com evidências. 
   ## Habilidades técnicas 

   **Testes manuais**: BVA · Classes de equivalência · Testes exploratórios · Cross-browser **Automação Web**: Selenium WebDriver · Python · Pytest · Page Object Model (POM) **Testes de API**: Postman (Collections, testes POST/DELETE, validação de status/body/contratos) **Bug reporting**: Jira · GitHub Issues **Banco de dados**: SQL (SELECT, JOIN, agregações) **Ferramentas**: Git · GitHub · Chrome DevTools · Android Studio (mobile básico) **Metodologia**: Metodologia ágil (Scrum) · Ciclo completo de QA
